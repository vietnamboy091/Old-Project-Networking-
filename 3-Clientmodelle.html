Die Clientmodelle

Detaillierte Beschreibung

Client-Server-Modell:

• Kommt für die Bereitstellung, Server-Diensten zum Einsatz
• Datenaustausch zwischen Client und Server über Protokolle ermöglicht
• Sorgt --> Aufgaben und Dienstlistungen auf verschiedene Rechner verteilt werden können
• Sind dauerhaft in Bereitschaft --> Zugriff der Clients jederzeit auf den Server
• Existieren weitere Konzepte --> Peer-to-Peer/ Master-Slave


Peer-to-Peer-Modell:

• Konzept zur Bereitstellung von Services
• Untereinander gleichberechtigt
• Können Server bereitstellen/ in Anspruch nehmen


Master-Slave-Modell:

• Konzept zur Regelung der Anwendungen oder Prozesse
• Übernimmt Steuerfunktionen/ erteilt ein/ mehrere Slaves das Recht --> kommunizieren/ Ressourcen zu nutzen


Typische Client/Server- Anwendungen und Protokolle::

• IT-Anwender basieren auf dem C.-S.-Modell
• Nutzen bestimmte Anwendungen oder Daten --> zur Verfügung gestellt von Anwendung-/Datenbankservern
• Viele Anwendungen nutzen das C.-S.-Modell
• Bekannteste --> World Wide Web
• Clients fragen Inhalte bestimmter Internetseiten an
• Kommunikation per http, HTTPS
• Webbrowser stellt HTTP, HTTPS Anfragen an Webserver --> erreichbar über IP-Adresse
• Server nimmt Anfrage entgegen, verarbeitet und liefert
• Im Webbrowser werden HTTP, HTTPS-Antworten verarbeitet, dem Anwender dargestellt


Vor- und Nachteile des Client-Server-Modells:

Vorteile:

• Zentrale Verwaltung der Services/ Ressourcen
• Sicheres zentrales Management der Zugriffsrechte
• kann viele Clients bedienen
• lassen sich per Netzwerke bereitstellen

Nachteile:

• Ausfall sorgt --> Dienstausfall- für kritische Dienste/ müssen Redundanzen bereitgestellt werden
• müssen mit genügend Bandbreite und Ressourcen ausgestattet sein
• Ressourcenengpässe verlangsamen die Antwort
• Betriebe und Bereitstellung ist mit erhöhtem Aufwand verbunden/ erfordert spezifisches Know-how


Typische Merkmale des Client-Server-Modells:

• Verteilung unterschiedlicher Aufgaben auf Client und Server
• Server stellen Dienste bereit
• Clients fragen Dienste bei Servern an
• Regelung der Kommunikation/ Informationsaustausches zwischen Client und Server
• bedienen ein oder mehrere Clients
• Server- und Clientfunktionen oder -aufgaben sind nicht an eine physische Hardware gebunden
• Physische Rechner können Client-/ Serveraufgaben ausführen
• Initiative zur Interaktion geht immer vom Client aus


Fat Client:

• Ein voll ausgestatteter, leistungsfähiger Desktop Computer
• Über ausreichend Rechenkapazität, Diskette/ CD-ROM-Laufwerk
• Plattenspeicher, leistungsstarke Grafikkarten verfügt
• Datenverarbeitung lokal auf dem Client --> eigene Festplatte, Laufwerk/ Programmierschnittstellen, der Hardware besteht
• Beispiel --> Windows-basierter PC


Thin Client:

• Computer/ Programm --> benötigt weniger Hardware und Rechenleistung
• Angewiesen auf den Server
• Betriebssystem und Programm-Updates werden vom Server bereitgestellt
• Wartungskosten, Rechenleistung und Energie kann gespart werden
• Beispiel --> Windows-Terminals, Netzwerkcomputer, NetPCs
• Viele nur in der Lage Daten herauszulesen/ Benutzereingaben an Server weiterzuleiten
